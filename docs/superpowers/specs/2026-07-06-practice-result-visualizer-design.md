## 一、我们已有什么(项目现状)

「数乡计划」是一个服务大学生三下乡社会实践的配套平台,目标是沉淀出一个可长期运营的**乡村数字资源库**——不是一次性的活动网站,而是一个会持续生长的乡村数据平台。50 名队员分 5 队赴湖北各村(样板村:武汉江夏小朱湾村)做实地实践,采集村庄资源、人物故事、影像、新闻稿,持续往平台填充数据。

### 技术底座

- **前端**:Vue3 + Vite 全套工程化重构(已放弃早期静态 HTML 框架),ECharts 全国地图分级下钻(全国 → 省 → 市 → 区县)。
- **测试与规范**:Vitest 单元测试覆盖数据校验、地图下钻、检索过滤等核心逻辑;设计文档(spec)与实现计划(plan)先行,测试基线保障不回归。
- **模块化架构**:一级栏目 = 一个独立模块,新增栏目只需三步(复制模板 → 登记配置 → 定义路由),老代码零改动,便于多人协作与长期迭代。

### 已建成 6 大内容栏目 + 首页中枢

| 栏目 | 作用 |
|------|------|
| **首页** | 品牌中枢 + 全国地图下钻可视化 + 六大栏目入口 + 乡村头条滚动 |
| **乡村百科** | 一村一页的数字名片:历史沿革、特色资源、人物、影像、荣誉认证;支持行政区划三级检索、荣誉标签筛选 |
| **乡村实践** | 汇聚三下乡团队的实践成果,用数据、人物、影像记录乡村的改变 |
| **乡村之声** | 乡镇发布真实需求,高校团队响应——「乡村出题,青年回应」的双向对接 |
| **实践攻略** | 行前准备、调研工具、报告模板、安全保障的一站式下乡指南 |
| **乡村好物** | 农副产品、手工艺、非遗与文创,以消费帮扶助力乡村振兴 |
| **关于我们** | 团队与愿景介绍 |

### 已有数据与视觉

乡村百科已有全国 12 个名村的富字段档案,首页地图接入湖北 6 村示例。全站暖绿大地色主题(米白 / 橄榄绿 / 珊瑚橘)、霞鹜文楷字体,风格温润克制、有人文气质。

**一句话:现有平台是一个内容比较全面、框架成熟的"展示型"网站,但内容基本是静态陈列的——看得到,却不会"动"、不会"帮你做事"。**

## 二、用户是谁、痛在哪

### 用户角色矩阵

| 角色 | 想要什么 | 现在的痛点 |
|------|---------|-----------|
| **实践队(核心用户)** | 策划有方向、执行不遗漏、成果拿得出手 | 选题没参照、过程没人盯、成果只能靠手搓 PPT,做得慢又难看 |
| **村庄 / 村干部** | 让"这支队带来的改变"被看见、留得下 | 成效说不清、留不下,队伍一走什么都没沉淀 |
| **后续队 / 评审老师 / 资助方** | 承接经验、对比成效、验证价值 | 成果散落在各队 PPT 和网盘里,无法汇聚对比 |

### 三个真实场景(痛点具象化)

- **策划期:两眼一抹黑。** 一支新队伍拿到"去小朱湾村做电商帮扶"的题目,却不知道往届有没有人做过类似的、村里到底有什么资源、该采集哪些数据才能证明成效。只能从零摸索,大量时间浪费在信息搜集上。
- **执行期:采着采着就散了。** 队员在村里忙活半个月,拍了几百张照片、记了一堆笔记,但没人系统盯着"方案里要的东西齐不齐"。回来整理时才发现关键的前后对比数据没采、某个人物访谈漏了,再也补不回来。
- **成果期:有料却讲不出。** 手上明明有调研报告、数据表、活动照片,但要在答辩台上讲清"我们到底带来了什么改变",只能连夜手搓 PPT,排版粗糙、数据干瘪,打动不了评委,更沉淀不到平台上供人对比。

**最痛的一条:三下乡"重形式、轻沉淀"——每年海量团队下乡,产出的成果要么套死模板千队一面,要么根本做不出能展示的东西。** 这正是我们要解决的问题。

## 三、我们要做什么

在「乡村实践」栏目下,做一个**贯穿一次实践从头到尾的 AI 实践智能体**。它不是又一个孤立栏目,而是把整个平台已有内容盘活的智能入口——用平台的真实数据帮实践队思考、生成、展示。

它有两个底层能力,贯穿始终:

- **懂平台**:优先用平台六大栏目的真实数据回答问题,给出可跳转的出处。
- **能补充**:平台里没有的,联网搜索兜底。

在此之上,分三个阶段陪伴一次实践:

### 阶段一 · 实践前(帮策划)

- 队员说出自己的 idea,AI 检索**全站已有资源**给建议:相似村庄档案、往届类似成果、可用的调研模板、乡镇真实需求,相关内容直接弹卡片、可跳转。
- 站内没有的,AI 联网搜索补充。
- 根据 idea 生成一份**实践方案初稿**:目标村、选题、计划要采集的指标、预期成果。
- 支持多模态输入:队员可上传文档、图片,和 AI 反复对话,持续丰富自己的方案。

> **举例**:队员输入"我们想去小朱湾村帮村民把土特产卖到网上"。AI 检索后回应:"平台里已有小朱湾村档案(点此查看),往届有 2 支队伍做过电商帮扶(参考成果),建议你重点采集这几个指标:帮扶前后月销量、参与农户数、村集体增收。已为你生成方案初稿,可继续补充。"

### 阶段二 · 实践中(督进度)

- 队员持续上传实地采集的数据、照片、访谈记录,材料自动归档。
- AI 对照最初的方案,分析已收集内容,**指出还缺什么、哪部分不足**,提醒队伍及时补齐。
- 为成果生成做实时准备,避免"回来才发现漏采"。

> **举例**:实践进行到第 10 天,AI 提示:"方案里计划采集的'帮扶后月销量'还没有数据;人物故事已采集 2 位,建议再补 1 位返乡青年的访谈,成果展示会更立体。"

### 阶段三 · 实践后(出成果)★ 重点

- AI 把采集回来的零散材料,**智能生成可直接用于答辩、展示、传播的可视化成果**。
- 同时提供**低代码 DIY 搭建台**:预置可视化组件与模板,队员可以像搭积木一样自己拖拽、增删、调整,拼出想要的成果作品。
- AI 生成打底 + 队员 DIY 微调——既智能省力,又完全可控,避免"AI 一键生成但改不动"和"纯手工搭建太累"两个极端。

## 四、成果可视化 · 具体方向(可执行清单)

这是项目最出彩的部分。我们不做"晒照片 + 堆数据"的老套展示,而是提供一批**能直接拿去答辩和传播**的可视化成品方向:

| 方向 | 是什么 | AI 在这里做什么 | 用在哪 |
|------|--------|----------------|--------|
| **AI 村庄导览图** | 交互式村庄地图,标注实践足迹、活动点位、帮扶成果,点击展开故事 | 从材料里提取地点与事件,自动布点、生成图注 | 答辩展示、村庄对外介绍 |
| **AI 成果海报** | 一键生成图文并茂的成果海报(标题 / 核心数据 / 亮点图) | 提炼核心数据与一句话亮点,自动排版配色 | 答辩、朋友圈 / 公众号传播 |
| **帮扶前后对比图** | 把"实践前 vs 实践后"的关键指标(销量 / 收入 / 参与户数)做成直观对比 | 对齐前后指标、算增长率、突出最亮眼的变化 | 最能打动人的"改变证明" |
| **实践足迹时间线** | 按时间串起走访、活动、里程碑事件 | 从记录中抽取时间与事件,自动排序成线 | 讲述实践全过程 |
| **人物故事墙** | 把采访到的乡土人物做成图文卡片墙 | 从访谈中提炼人物身份、金句、故事梗概 | 有温度的成果呈现 |
| **成果数据看板** | 走访户数 / 活动场次 / 覆盖人数等指标的仪表盘 | 从表格 / 报告中抽取指标数字,汇总成看板 | 一眼看清实践体量 |
| **一村一实践长图** | 把上述元素整合成一张可长图导出的完整成果页 | 智能编排各模块顺序与布局 | 汇报、存档、分享 |
| **宣传文案 / 短视频脚本** | AI 把成果数据翻译成可传播的文字与脚本 | 把数据"翻译"成有故事感的文案 | 二次传播 |

**使用方式有三档,照顾不同需求:**

1. **全自动**:队员上传材料,AI 直接推荐并生成一套完整成果页,一键可用。
2. **意图驱动**:队员说一句"我想重点突出帮扶后收入翻倍",AI 据此编排出对应的可视化。
3. **完全 DIY**:队员进搭建台,自己从组件库拖拽拼装,AI 只负责填充数据和润色文案。

## 五、创新点(我们和普通展示网站的不同)

### 1. 平台知识激活 —— 把"静态网站"变成"会思考的系统"

普通乡村网站只是把内容陈列出来给人看。我们的 AI 智能体把网站六大栏目当成自己的知识库,能检索、能引用、能给出可跳转出处,用平台的真实数据帮用户做事。**同样一个网站,别人做成了"电子画册",我们做成了"能对话、能生成的智能平台"** —— 这是应用层面的差异化。

### 2. 一次实践全程陪伴 —— 从"割裂工具"到"完整生产线"

市面上的工具要么只帮找资料(策划),要么只帮做图表(成果),彼此割裂。我们用一份**贯穿始终的"实践档案"**把三阶段串成一条线:策划期生成的方案,执行期照着它采集、查缺,成果期直接拿它生成可视化。**数据从头贯穿到尾,不是三个功能的拼盘,而是一条完整的实践生产线。**

### 3. 意图驱动的生成式可视化 + DIY 搭建 —— 破解"千队一面"与"门槛太高"

- 传统模板工具:选一个死模板套进去,千队一面,不贴合自己的实践。
- 专业设计工具:能做出好东西,但学习门槛高,队员搞不定。
- **我们的做法**:队员用一句话说出想强调什么,AI 智能编排出贴合自己材料的可视化作品(导览图 / 海报 / 对比图…),再让队员低代码 DIY 微调。**既智能又可控,针对三下乡成果的这种工具目前没有对标产品。**

## 六、分阶段落地

- **第一阶段(实训期)**:三阶段全部打通成可用闭环,**开发重心放在"成果可视化"**——先把导览图、成果海报、帮扶前后对比等几个方向做扎实;实践前 / 实践中做成轻量可用,串起完整故事。这一阶段目标是拿出一个能完整演示的作品。
- **第二阶段(比赛期)**:把可视化生成能力做深做炫——扩充更多成果形态、增强 AI 的智能编排与多模态理解,作为软件创新大赛冲刺的技术亮点。

## 七、技术选型(简述)

- **AI 能力**:基于 DeepSeek / 阿里云百炼 API,含多模态(理解照片)与文本生成,成本可控、有免费额度;后续可平滑升级更强模型。
- **前端**:延续现有 Vue3 + ECharts,可视化落在「乡村实践」栏目新增页面,复用全站暖绿主题。
- **后端**:Node + Express + SQLite + 本地文件存储(延续既定后端化方案),支持队员登录、在线上传材料、账号可追溯。
- **工程规范**:延续"设计先行 + 测试驱动 + 核心逻辑人工可讲解"的既有规范,AI 生成代码经人工审查修正。
