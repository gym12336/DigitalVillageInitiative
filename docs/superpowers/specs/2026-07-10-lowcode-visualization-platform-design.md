## 一、背景与定位

实践队每年采集大量调研数据、访谈、照片、报告，却缺工具把这些原始材料快速整理成可展示、可传播的成果，最终往往只剩一份排版粗糙的 Word/PPT。成果可视化助手聚焦「实践后·出成果」这一段。

**本次相较原 PRD 的方向调整**：不再做单一的「左AI右3D地图」固定工作台，而是做一个**低代码可视化平台**——工作台 = 画布 + AI 助手 + 组件，分两级：

1. **工作台一 · 大组件制作台**：用基础组件在栅格画布上拼出可复用的「大组件」（如实践时间轴），存入大组件库。
2. **工作台二 · 成果搭建台**：把大组件 + 基础组件拖进画布，选一个村子的实践档案作为数据源，拼成一份可视化成果作品，导出 JSON 或导出静态展示网站。

原「3D 地图」不再是锁死的工作台，而是平台里的一个「地图大组件」，与「时间轴」「对比图」在架构上平等。

**本期范围（明确收敛）**：先把**低代码引擎骨架 + 端到端闭环**做扎实（能拼、能存、能导出），AI 能力预留接缝、本期不实现。详见第八节。

---

## 二、六个地基决策（已确认）

本设计建立在以下六个已拍板的决策上：

1. **两个工作台都是面向实践队员的完整低代码环境**（非「工作台一仅开发者用」）。
2. **画布布局范式 = 栅格**（组件拖入栅格、可跨列跨行、可调宽；导出网站天然响应式）。
3. **工作台外壳 = 三竖栏「组件+AI 在左 · 画布居中 · 属性面板常驻右」**。
4. **大组件 = 数据插槽模型（模板与数据分离）**：大组件只定结构 + 插槽声明，不填死数据；使用时插槽绑定数据源灌值。
5. **插槽绑定到现有实践档案数据结构**：复用 `dossier.collected` 已有词汇（`metricValues / materials / people / places / photos` 等），不新造数据模型。
6. **导出 = JSON（免费副产品，做）+ 静态 HTML 包（给人看的成果，做）**。

## 三、整体架构与分层

核心思想：**一份成果 = 一棵 JSON（布局 + 组件 + 数据绑定）；两个工作台都在编辑这棵 JSON；渲染引擎把它画出来——工作台内用 Vue 交互式渲染，导出时用同一套逻辑生成纯静态页。**

四层，从下到上：

```
① 内核层（纯函数，禁飞区，全 TDD，不依赖 Vue）
   registry  —— 组件白名单：基础组件 + 大组件各需要哪些插槽/属性/可绑数据源
   validate  —— 校验一棵作品 JSON 落在 registry 约束内
   binding   —— 数据插槽 → 实践档案字段 的解析（把村子数据灌进插槽）
   renderer  —— 遍历作品 JSON，产出「渲染描述」（与 Vue 无关的纯数据）

② 数据层
   复用已有实践档案 dossier（metricValues / materials / people / places / photos）
   新增 works 表存作品 JSON、component_defs 表存大组件定义（SQLite，延续既定后端化）

③ 工作台层（Vue，消费内核）
   共享外壳 StudioShell：组件面板 + AI 助手（左） / 栅格画布（中） / 属性面板（右）
   工作台一 ComponentStudio —— 造大组件（定结构 + 插槽），存入大组件库
   工作台二 ResultStudio    —— 拼成果（大组件 + 基础组件 + 选村灌数据）

④ 导出层
   toJSON       —— 作品对象原样序列化（免费）
   toStaticSite —— 把渲染描述 + 数据打包成自带 HTML/CSS/JS 的静态包
```

**关键取舍**：内核层为纯函数、不依赖 Vue，所以同一棵 JSON 既能在工作台里用 Vue 交互式渲染，也能导出成脱离平台的静态页——这就是「静态 HTML 包」能成立、又不必重复实现渲染逻辑的根本。已废弃地图工作台里值钱的 `schema/validate/renderer` 内核，原样降级为「地图大组件」的内部实现并入本层。

**依赖方向**：`registry ← validate`、`registry ← binding`、`registry + binding ← renderer`、`工作台/导出层 → 内核`。无循环依赖。

## 四、两个工作台的形态与共享外壳

**共享外壳（外壳 B，两个工作台复用同一个 `StudioShell.vue`）：**

```
┌─────────────┬────────────────────────┬──────────────┐
│ 左栏         │  中栏 · 栅格画布         │ 右栏         │
│ 上：组件面板 │  拖组件进格子            │ 属性面板     │
│ 下：AI 助手  │  可跨列/跨行/调宽        │ （选中组件后 │
│ （对话框）   │  点击选中                │  改标题/颜色/│
│             │                        │  数据绑定）  │
└─────────────┴────────────────────────┴──────────────┘
   顶部工具条：保存 · 导出 JSON · 导出网站 · 预览
```

外壳把「拖拽、栅格布局、选中、属性编辑、保存」这些通用能力写一遍，通过两个差异点配置成两个工作台：**组件面板里放什么** + **AI 助手干什么活**。

**工作台一 · 大组件制作台（ComponentStudio）：**
- 左栏组件面板放**基础组件**（文本、图片、数据条、图表原子、地图点位）。
- 画布上拼出复合结构，并给需要动态数据的地方**开插槽**（如时间轴的「事件列表」槽）。
- 属性面板里设「这个插槽绑到实践档案的哪个字段」（如 `materials`）。
- AI 助手（本期预留接缝）：帮生成结构骨架、起名、写占位文案。
- 产出：一个**大组件定义**（结构 + 插槽声明），存入大组件库（`component_defs` 表）。

**工作台二 · 成果搭建台（ResultStudio）：**
- 左栏组件面板放**大组件库**（时间轴、对比图、人物墙、地图、KPI…）+ **基础组件**（标点弹窗卡、文本、图片）。
- 顶部先**选一个村 / 一份实践档案**作为数据源。
- 拖大组件进画布，插槽**自动从选定档案灌数据**；AI 助手（本期预留接缝）帮布局和填充。
- 属性面板改文字、换绑定、调样式。
- 产出：一份**成果作品**（布局 + 组件实例 + 数据绑定），保存（`works` 表）/ 导出。

**关键点**：两个工作台产出的 JSON 用**同一套 registry/validate 校验**。工作台一产出「可复用的大组件定义」，工作台二产出「填好数据的成果实例」；大组件定义被工作台二引用时，相当于一个「带插槽的子树」嵌进成果树里。

## 五、内核四个纯函数模块（禁飞区）

内核层是整个平台的「AI 禁飞区」——纯 JS、不依赖 Vue、全 TDD、可逐行讲解。住在 `DigitalVillageInitiative/src/modules/practice/lowcode/`。

**① `registry.js` — 组件白名单（约束边界）**
- 声明每种组件的契约：基础组件（`text / image / metricBar / chartAtom / mapPoint` 等）和大组件各自需要哪些**属性**、开哪些**插槽**、插槽能绑哪些**数据源**（复用实践档案词汇 `metrics / before_after / materials / people / places / photos`）。
- AI、校验器、渲染器三方唯一契约来源。改这里 = 增减平台能力。无逻辑，纯声明。

**② `validate.js` — 作品 JSON 校验（禁飞区）**
- `validateWork(work)` → `{ valid, errors }`。检查：布局合法（栅格坐标合规）、每个组件类型在 registry 内、必填属性齐全、插槽绑定的数据源在白名单内、大组件引用存在。
- 渲染前防线 + 防 AI 脏输出/注入的安全闸。只读 registry，不依赖 renderer。

**③ `binding.js` — 数据插槽解析（禁飞区）**
- `resolveBindings(work, dossier)`：遍历作品里每个插槽，按绑定字段从实践档案取数据灌入，产出「已填充数据的作品树」。
- 取不到数据时落空占位 + 标记「待补充」，不静默出错。这是「实践中上传 → 实践后成果」数据流的接缝。

**④ `renderer.js` — 渲染描述生成（禁飞区）**
- `renderWork(filledWork)`：遍历已填充作品树，产出一棵**与 Vue 无关的「渲染描述」**（每个组件：类型、栅格位置、尺寸、最终属性、数据）。
- 输出是纯数据，不是 Vue 组件也不是 HTML 字符串。工作台里由 Vue 消费它交互式渲染；导出层拿同一棵渲染描述生成静态 HTML。**一套渲染逻辑，两个消费端。**
- 地图大组件的 render 分支内部调用已废弃地图工作台降级并入的 `schema/validate/renderer` 内核。

全部模块可脱离服务器和 AI 独立测试与演示。

## 六、AI 层与异常处理

**AI 定位**：AI 是「打底」，内核是「闸门」——AI 一律产出「受约束的 JSON」，先过 `validateWork` 才落地，绝不直接产渲染代码。

**本期处理**：内核和工作台里**预留 AI 接缝**（`composeWork` / `fillCopy` / `suggestStructure` 的调用位，以及「AI 产出必须过 validateWork」的闸门），**但不实现具体 AI 能力**。等骨架跑通再回来决定接什么、接多深。后续规划的 AI 能力方向（见第八节）：

- 工作台一 `suggestStructure`：一句话 → 大组件结构草稿（含插槽声明）。
- 工作台二 `composeWork`：一句话意图 → 从库里挑组件、编排布局、绑数据源，产出成果草稿。
- `fillCopy`：从实践档案数据润色文案（标题、图注、导语、亮点）。
- 接法：复用现有 `server/services/deepseek.js`（OpenAI 兼容、强制 JSON、令牌桶限流、失败重试、无 key 兜底），新增 `server/services/lowcodeAiService.js` 封装。

**异常处理（延续既有兜底）：**
- **AI 无 key / 超时 / 失败**（AI 接入后）：明确提示，保留画布已有内容不丢失，允许重试或纯手动继续。
- **非法作品 JSON**：`validateWork` 拦下，不渲染坏组件，提示「这部分没生成好，已跳过」，队员手动补。
- **数据插槽取不到值**：`binding` 落空占位 + 标「待补充」高亮，引导队员回实践档案补数据，而非静默出空。
- **大组件引用丢失**（库里被删）：校验报错并提示，不渲染悬空引用。
- **导出**：导出前强制跑一次 `validateWork`，不合法则拦住并提示，避免导出坏页面。

## 七、测试策略

延续项目 Vitest + TDD 基线，勿回归。

- `registry / validate / binding / renderer` 四个纯函数模块全 TDD 覆盖（合法/非法作品、栅格坐标校验、插槽绑定取值与兜底、各组件渲染描述、大组件引用与嵌套）。
- 端到端集成测试：一份样例实践档案 + 一棵手写作品 JSON → validate → binding → renderer → 渲染描述一条链跑通；再走 `toStaticSite` 产出静态页断言。
- 后端 service / 导出层测试用依赖注入的假 fetch / 假档案，不打真 API。

## 八、本期范围收敛与里程碑

**策略**：两个面向队员的完整低代码工作台是个大摊子。本期**先把骨架和端到端闭环做扎实**（能拼、能存、能导出），AI 放到骨架之上、后续再研究。宁可一条闭环做透，不要多个半成品。

**本期端到端闭环（骨架，不含 AI，须能连续演示）：**
```
选一份实践档案（实践中已上传的数据）
   → 工作台二：从组件库手动拖大组件 + 基础组件进栅格画布
   → 插槽自动从档案灌数据 + DIY 微调（改文字/换绑定/调样式）
   → 保存
   → 导出 JSON + 导出静态网站
```

**本期做 / 缓做：**

| 能力 | 本期 | 说明 |
| --- | --- | --- |
| 内核四个纯函数模块（registry/validate/binding/renderer） | ✅ 做透，全 TDD | 禁飞区，答辩逐行讲 |
| 工作台二（拼成果）端到端闭环 | ✅ 做透 | 演示主线 |
| 一批**预置大组件**（时间轴/对比图/人物墙/地图/KPI） | ✅ 做 | 由团队用工作台一预先造好，保证工作台二有料可拼 |
| 工作台一（队员从零造大组件） | ⚠️ 做骨架 | 能造、能存、能开插槽；防呆/复杂交互二期补 |
| 导出 JSON | ✅ 做 | 免费副产品 |
| 导出静态 HTML 包 | ✅ 做 | 「沉淀可传播数字资产」的体现 |
| AI 能力（compose/fill/suggest） | 🔸 只预留接缝 | 骨架跑通后再决定接什么、接多深 |
| 平台知识 RAG（检索六大栏目） | ⏭ 二期 | 「会思考的系统」叙事，本期不做 |

**里程碑：**
- **本期（实训期）**：低代码骨架 + 工作台二闭环 + 预置大组件 + 双导出，`npm test` 全绿。拿出能完整演示的作品。
- **二期（比赛期）**：接入并做深 AI（意图编排、文案填充、结构建议），补工作台一防呆，扩更多组件形态，做 RAG 窄 demo。

## 九、验收标准（本期）

1. 能在工作台一拼出一个带插槽的大组件并存入库（骨架级）。
2. 能在工作台二选一份实践档案、从库里拖大组件 + 基础组件进栅格画布，插槽自动灌入该档案数据。
3. 能 DIY 微调（改文字、换绑定、调样式、增删组件），保存后作品持久化、可再编辑。
4. 能导出作品 JSON；能导出一个双击即可在浏览器打开的静态 HTML 包，脱离平台正常展示。
5. 非法作品被 `validateWork` 拦下、插槽取不到值有「待补充」兜底、大组件悬空引用被拦。
6. `npm test` 全绿，内核四个纯函数模块 + 端到端集成测试覆盖，无回归。

## 十、未来方向（本期不做）

- **AI 三档能力**：填充型（fillCopy）→ 编排型（composeWork，意图驱动生成式可视化，核心亮点）→ 平台知识 RAG（把静态网站变成会思考的系统）。
- **工作台一防呆化**：让非专业队员也能稳定从零造大组件。
- **更多组件形态与多模态**：照片内容理解、更丰富的图元/图表原子。
- **实践前/中两段串联**：与本工作台连成贯穿三阶段的完整实践生产线。
