# 数乡计划 · 乡村数字资源库平台 — 设计文档

- 文档日期：2026-07-02
- 状态：已确认，待生成实施计划
- 适用范围：平台首版（脚手架 + 首页中枢 + 3D 地图 + 村庄主页模板 + 模块注册机制）

---

## 1. 背景与目标

「数乡计划」是一个大学生社会实践项目，最终产出一个可长期运营的「乡村数字资源库」平台，而非一次性网站。50 名队员分 5 队赴湖北各村（样板村：武汉江夏小朱湾村）实地采集村庄资源、人物故事、影像、新闻稿等，持续往平台填充数据。

平台核心栏目（源自实践执行方案技术开发组职责）：网站首页、村庄列表、村庄主页模板、地图/榜单可视化。

**本设计目标**：搭建一个模块化、可扩展的平台基础，首页作为「导航中枢」，以一个 3D 全国地图下钻作为视觉焦点与村庄数据的空间入口。为后续持续增加功能模块、由更多人手（含大模型协助）开发打好工程基础。

**成功标准**：
- 能运行的 Vue3 + Vite 工程，首页展示 3D 地图并可下钻到区县级
- 首页从模块清单自动生成功能入口卡片
- 存在新模块脚手架，新增模块无需改动既有代码
- 有一个可用的村庄主页模板，用示例数据（小朱湾村等）跑通「地图点位 → 村庄主页」的完整链路

---

## 2. 技术选型

| 项 | 选择 | 理由 |
|---|---|---|
| 框架 | **Vue3 + Vite** | 面向长期平台化，组件化 + 工程化基础，团队扩员与大模型协作友好 |
| 路由 | **Vue Router** | 多页面（首页/列表/村庄详情/各模块） |
| 3D 地图 | **ECharts-GL**（`echarts` + `echarts-gl`） | 无需自建 WebGL，支持 3D 立体地图（map3D）、下钻、旋转、散点，视觉可达大屏科技风 |
| 数据 | **本地 JSON**（`src/data/`） | 首版无后端；结构预留将来切换后端 API 的空间 |
| 地图底图 | **geoJSON**（`src/assets/geo/`） | 全国/省/市/区县行政边界，按需加载 |

> 决策记录：曾评估「保留原静态 HTML + CDN 引入 ECharts」方案（改动小、维护门槛低）。用户明确选择 **Vue3 全套重构（B 方案）**，理由是面向长期复杂开发、要打好工程基础、后续会增加人手并借助大模型协助。原静态框架（`DigitalVillageInitiative/`）作为内容与视觉参考保留，不再作为运行代码。

## 3. 整体架构

### 3.1 首页 = 导航中枢（Hub）

首页不堆砌功能，只承担两件事：

1. **上半屏**：3D 全国地图（视觉焦点 + 村庄空间入口）
2. **下半屏**：功能模块入口卡片列表，**从模块清单自动生成**

点击卡片进入对应模块的独立页面。功能分散到各模块，保持首页清晰。

### 3.2 目录结构

```
shuxiang-platform/
├─ index.html                  # Vite 入口
├─ vite.config.js
├─ package.json
└─ src/
   ├─ main.js                  # 挂载 Vue + Router
   ├─ App.vue
   ├─ modules.config.js        # ⭐ 模块清单（唯一的模块登记处）
   ├─ router/
   │  └─ index.js              # 静态路由 + 自动收集各模块路由
   ├─ views/
   │  └─ HomeView.vue          # 首页中枢（读清单渲染入口卡）
   ├─ components/
   │  ├─ ChinaMap3D.vue        # ⭐ 3D 下钻地图（核心组件）
   │  ├─ SiteHeader.vue
   │  ├─ SiteFooter.vue
   │  └─ ModuleCard.vue        # 首页模块入口卡
   ├─ modules/                 # 每个模块一个独立文件夹
   │  ├─ villages/             # 村庄主页 + 村庄列表
   │  │  ├─ routes.js
   │  │  ├─ VillageListView.vue
   │  │  └─ VillageDetailView.vue
   │  ├─ ranking/              # 资源榜单（首版可仅占位）
   │  ├─ people/               # 人物故事（首版可仅占位）
   │  ├─ media/                # 影像库（首版可仅占位）
   │  └─ _template/            # ⭐ 新模块脚手架模板
   │     ├─ routes.js
   │     └─ IndexView.vue
   ├─ data/
   │  ├─ villages.json         # 村庄数据（核心字段 + 扩展区）
   │  └─ ...
   └─ assets/
      ├─ geo/                  # 省市区县 geoJSON
      └─ theme/                # 大地金主题变量
```

### 3.3 模块化可插拔机制

**模块清单 `modules.config.js`** 是新增模块的唯一登记处：

```js
export const modules = [
  { id: 'villages', name: '村庄主页', icon: '🏘️', path: '/villages', enabled: true },
  { id: 'ranking',  name: '资源榜单', icon: '🏆', path: '/ranking',  enabled: true },
  { id: 'people',   name: '人物故事', icon: '👤', path: '/people',   enabled: true },
  { id: 'media',    name: '影像库',   icon: '🎬', path: '/media',    enabled: true },
]
```

- **首页**读取该清单，为每个 `enabled` 模块渲染一张入口卡片（`ModuleCard.vue`）。
- **路由**在 `router/index.js` 中通过 `import.meta.glob` 自动收集各 `modules/*/routes.js` 并注册，无需手工维护路由表。
- `enabled: false` 的模块可在首页隐藏但保留代码，用于「尚未想好、先占位」的模块。

**新增一个模块 = 3 步**（老代码零改动）：
1. 复制 `modules/_template/` → 重命名为新模块目录
2. 在 `modules.config.js` 增加一条记录
3. 完成 —— 首页自动出现入口卡，路由自动生效

## 4. ChinaMap3D 组件设计

### 4.1 职责

- 用 ECharts-GL 渲染 3D 立体行政地图（省份有厚度、可旋转、发光边框）
- 逐级下钻：**全国 → 省 → 市 → 区县**
- 面包屑导航返回上级（如「全国 / 湖北省 / 武汉市 / 江夏区」）
- 在当前层级地图上以发光散点标记村庄点位
- 点击村庄散点 → 路由跳转到该村庄主页

### 4.2 下钻的递归分级设计（关键：为 C 阶段升级预留）

下钻逻辑设计为**递归分级**，每一级的行为由数据驱动而非写死：

- 每一级持有：当前行政区编码（adcode）、层级类型（country/province/city/district）、对应 geoJSON
- 点击某区域 → 加载下一级 geoJSON → 重绘 → 压入面包屑栈
- **首版（B）**：下钻到 **区县级** 为止；村庄以散点标注在区县地图上，点散点跳村庄主页。
- **升级路径（C，本设计预留、暂不实现）**：在区县这一级增加「点击区县 → 展开该区县下的村庄点位/列表」的下一层。由于下钻是递归分级的，C 只是在末级追加一层「村庄层」，**无需重写组件**。

### 4.3 geoJSON 数据加载策略

- 全国、湖北省、以及涉及样板村/候选村的市与区县 geoJSON 按需加载（点击下钻时才拉取），避免首屏加载全部区县数据。
- geoJSON 放 `src/assets/geo/`，首版内置项目涉及到的湖北相关行政区；其余省市可后续补充。
- 数据来源可用公开行政区划 geoJSON（如阿里 DataV.GeoAtlas 等），实施阶段确认许可与准确性。

### 4.4 视觉风格：大地金（B）

- 底色：暖褐/深褐渐变背景
- 地图块：金橙渐变，发光边框（金色辉光）
- 散点：金色发光点，hover 放大 + tooltip 显示村名
- 主题变量集中在 `assets/theme/`，便于统一微调
- 呼应「数乡·乡土·丰收」主题，区别于常见的科技蓝数据大屏

## 5. 数据设计（C 策略：字段表 + 示例数据）

采用「先定义可扩展字段表，同时用符合结构的示例数据跑通效果」。字段表为**第一版、可扩展**，预调研阶段会验证并调整（符合实践执行方案「用 1 个村庄测试模板，确认哪些字段需修改」的安排）。

### 5.1 设计原则

- **核心字段（稳定）**：地图与列表强依赖，结构尽量不变。
- **扩展内容区（灵活）**：不确定的内容放这里，增删字段不影响地图/列表运行。
- 字段变更只改 `src/data/`，不改组件。

### 5.2 村庄数据结构 `villages.json`

```jsonc
{
  "id": "xiaozhuwan",              // 唯一标识（核心）
  "name": "小朱湾村",               // 村名（核心）
  "fullName": "武汉市江夏区小朱湾村", // 全称（核心）
  "province": "湖北省",             // 省（核心，用于地图归属）
  "city": "武汉市",                 // 市（核心）
  "district": "江夏区",             // 区县（核心，村庄点位落在此级）
  "adcode": "420115",              // 区县行政编码（核心，匹配 geoJSON）
  "coord": [114.32, 30.37],        // 经纬度 [lng, lat]（核心，地图散点）
  "type": "文旅基础",               // 村庄类型（核心，用于筛选/配色）
  "summary": "近郊乡村数字主页样板", // 一句话简介（核心，卡片/tooltip）
  "cover": "",                     // 封面图路径（核心，可为空占位）
  "status": "样板试点",             // 状态标签（核心）

  "extra": {                       // 扩展内容区（灵活，随实践补充）
    "history": [],                 // 村史时间线
    "people": [],                  // 人物故事
    "resources": [],               // 特色资源卡片
    "media": [],                   // 照片/视频
    "route": [],                   // 导览路线
    "outcomes": []                 // 实践成果
  }
}
```

### 5.3 首版示例数据

以样板村小朱湾村为主，加入实践执行方案中的候选村（大余湾、刘家桥、羊楼洞、彭家寨、杏花村、高峰村等）作为示例点位，核心字段填真实可查信息（省市区、大致经纬度、类型），`extra` 留空或少量占位。用于跑通地图下钻与点位跳转，实地采集后替换。

## 6. 数据流

```
villages.json ──┬─→ ChinaMap3D（按 province/adcode/coord 生成散点）
                │        └─ 点击散点 → router.push(/villages/:id)
                ├─→ VillageListView（列表/筛选）
                └─→ VillageDetailView（读 :id 渲染核心字段 + extra 各栏目）

modules.config.js ──┬─→ HomeView（渲染模块入口卡）
                    └─→ router（配合各 modules/*/routes.js 注册路由）
```

## 7. 错误处理与健壮性

- **geoJSON 加载失败**：显示占位提示并保留在当前层级，不崩溃；控制台记录失败的 adcode。
- **村庄缺少经纬度/adcode**：该村不在地图上打点，但仍出现在村庄列表中（核心字段中 coord 缺失时降级处理）。
- **扩展区为空**：村庄主页对应栏目显示「待实地采集补充」占位，不报错。
- **模块 enabled=false 或路由缺失**：首页不渲染其卡片；直接访问其路径时给出友好提示。

## 8. 测试策略

- **组件级**：`ChinaMap3D` 的下钻状态机（进入下一级 / 面包屑回退 / 到达区县末级）用单元测试覆盖；数据映射函数（villages.json → 散点数组）单测。
- **数据校验**：villages.json 核心字段完整性校验脚本（缺字段时报出是哪个村的哪个字段）。
- **路由/模块注册**：验证 modules.config.js 中每个 enabled 模块都能解析到对应路由。
- **手动验收**：浏览器中跑通「全国→湖北→武汉→江夏 下钻 → 点小朱湾散点 → 进入村庄主页」完整链路。
- 测试框架在实施计划阶段选定（Vitest 为 Vite 生态默认选择）。

## 9. 首版范围（YAGNI）

**做**：
- Vue3 + Vite 工程脚手架
- 首页中枢（3D 地图 + 模块入口卡自动生成）
- ChinaMap3D：下钻到区县级 + 示例村庄散点 + 点位跳转
- 一个可用的村庄主页模板（VillageDetailView）+ 村庄列表
- 模块注册机制 + `_template/` 脚手架
- 大地金主题
- 示例数据（小朱湾 + 候选村）

**暂不做（留入口/占位）**：
- 后端 / 数据库 / 登录鉴权
- 资源榜单、人物故事、影像库等模块的内部完整功能（先占位入口）
- C 阶段的「区县 → 村庄点位」下钻层（架构预留，暂不实现）
- 真实影像/图片资源（实地采集后填充）
