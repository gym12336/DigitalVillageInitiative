# 搭建台组件库分类重组 设计

## 问题

1. `type: 'image'` 组件出现在"封面开场"和"媒体嵌入"两个分类中，功能完全相同，造成冗余
2. "荣誉佐证"分类下只有一个通用"数据表"组件，语义不匹配
3. "封面开场"和"媒体嵌入"使用相同图标 🎬，容易混淆

## 方案

### 变更

| 操作 | 内容 |
|------|------|
| 合并分类 | `🎬 封面开场` + `🎬 媒体嵌入` → `🖼️ 媒体图片`（含封面大图 + 图片两项） |
| 迁移组件 | `📋 数据表` 从 `🏆 荣誉佐证` → `🍩 讲「构成 / 分布」`（与饼图、堆叠柱并列） |
| 删除分类 | `🏆 荣誉佐证`（将来有真正荣誉类组件再加回来） |

### 最终分类树（12 个，精简自原 14 个）

```
📊 讲「变化」— 帮扶对比      哑铃图、涨跌徽标
📈 讲「整体画像」— 概览      KPI 卡组、雷达图
⏱️ 讲「过程」— 时间线        时间轴
🍩 讲「构成 / 分布」         饼图、堆叠柱、数据表
👥 讲「人与故事」            人物卡、金句块
🗺️ 讲「空间」— 地理分布      地图散点
🖼️ 媒体图片                 封面大图、图片
🔢 单点强调                 大数字
🔀 关系流向                 桑基图
📅 时间频率                 日历热力
🔄 交互对比                 前后对比
📦 组合                     多组件框
```

### 涉及文件

| 文件 | 变更 |
|------|------|
| `src/modules/builder/editor/ComponentLibrary.vue` | 替换 cover + media 分类为合并后的 media 分类，datatable 移至 composition，删除 honor |
| `src/modules/builder/display/DisplayComponentLibrary.vue` | 同上 |

纯配置修改（数组结构调整），不改组件逻辑、渲染、属性面板。
