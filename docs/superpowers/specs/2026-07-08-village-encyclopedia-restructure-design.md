# 乡村百科板块重构设计

- 作者：gym

> 日期：2026-07-08　范围：DigitalVillageInitiative 前端「乡村百科」板块

## 一、背景与问题

「乡村百科」当前结构混乱、子目录跳转不准。根因有四：

1. **数据源分裂。** 主页面「村庄名录」`/villages` 用 `encyclopedia-villages.json`（12 个全国名村，数据丰富）；三个子页「乡村人物」`/people`、「影像记录」`/media`、「乡村榜单」`/ranking` 用另一套 `villages.json`（6 个湖北实调村）。两套村庄互不相通——在名录点进某全国名村，切到子页却看到另外 6 个村，这是"跳转不准"的直接根源。
2. **导航 path 撞车。** 子目录「村庄名录」的 path (`/villages`) 与父栏目「乡村百科」(`/villages`) 完全相同，层级形同虚设。
3. **子页名不副实。** 「乡村榜单」标题实为"乡村特色资源"，内容是按类型列资源，并非榜单。
4. **内容充实度不均。** 名录页 + 详情页（8 模块）完整，人物/影像/榜单三个子页很薄（78–108 行），多为"待采集"占位空壳。

## 二、关键约束：12 村的真实数据画像

12 个全国名村（`encyclopedia-villages.json`）**12 村全覆盖**的字段：`honors`、`tags`（六大类：文化/自然/产业/荣誉/人文/新业态）、`gallery`（实景图）、`guide`（导览点位带讲解）、`stats`（views/favorites/practices）、`socials`、`intro`、`summary`、`certLevel`、`manager`、`coord`。

**完全没有**的字段：`people`（人物）、`media`（视频）、`resources`（资源清单）——这三个字段只存在于将被淘汰的 6 村老数据里。

设计据此裁剪：不为填坑造假数据，只用真实字段撑起横向页；人物作为详情页的占位录入模块。

## 三、核心决策

| 项 | 决策 |
|---|---|
| 数据源 | 全站统一到 `encyclopedia-villages.json`（12 村），淘汰 `villages.json` |
| 板块定位 | 村庄百科，以村为中心（一村一页为核心） |
| 结构方案 | 一主三辅：村庄名录（主）+ 荣誉榜单 + 影像长廊 + 分类浏览 |
| 详情页 | 按「感受 → 了解 → 行动」重排，人物作为占位录入模块 |
| 人物 | 无独立子页；作为详情页模块，12 村暂无数据时显示引导录入占位入口 |

## 四、导航与路由结构

```
📖 乡村百科（父，悬浮弹下拉，点击进名录）
   ├ 村庄名录   /villages          （= 父页本身，下拉不重复列）
   ├ 荣誉榜单   /villages/honors
   ├ 影像长廊   /villages/gallery
   ├ 分类浏览   /villages/tags
   └ 一村一页   /villages/:id       （详情，从卡片/榜单/图墙进入）
```

- 修掉 path 撞车：父栏目点击进 `/villages`；下拉子项为三个新横向页，不再重复列"村庄名录"。
- `modules.config.js` 的 `children` 相应更新为 honors/gallery/tags 三项。
- 旧路由 `/people`、`/media`、`/ranking` 保留并 **重定向到对应新页**（people → /villages；media → /villages/gallery；ranking → /villages/honors），避免外部链接 404。

## 五、页面职责与排版

### 5.1 村庄名录 `/villages`（现有，基本不动）
头条轮播 + 搜索 + 排序胶囊 + 行政树级联 + 荣誉墙 + 卡片网格。仅需确认其数据源已是 `encyclopedia-villages.json`（本就如此）。

### 5.2 一村一页详情 `/villages/:id`（重排）
按叙事顺序重排 8 模块：

- **感受**：① 大图轮播 `gallery` → ② 村名 · 地点 · 荣誉徽章（认证/负责人/社媒压缩为村名下方一行小信息条，不再单列夹中间）
- **了解**：③ 村庄详述 `intro` → ④ 六大类标签 `tags`（点标签跳 `/villages/tags`）→ ⑤ 导览点位 `guide` → ⑥ **乡村人物**（新模块）
- **关联 & 行动**：⑦ 该村实践成果 → ⑧ 底部工具条（生成导览图/海报/分享，去掉与⑦重复的"看成果"按钮）→ 浮动点赞/收藏（保留）

**人物模块**：读取村庄 `people` 字段（暂不存在）。无数据时显示引导录入占位入口（提示"人物故事待实地采集录入"+ 录入引导），不隐藏、不空壳。

### 5.3 荣誉榜单 `/villages/honors`（替换假"特色资源"）
- 顶部维度切换胶囊：最热浏览 / 最多收藏 / 最多实践（读 `stats`）。
- 主体榜单行：名次徽章 · 村名 · 地点 · 荣誉标签 · 该维度数值条形。点行进详情。
- 可选"荣誉之最"小卡（如唯一的中国历史文化名村）。

### 5.4 影像长廊 `/villages/gallery`（原"影像记录"，改用真图）
- 聚合 12 村 `gallery` 实景图，masonry/网格图墙，图上叠村名。
- 顶部按村 / 按标签类别筛选；hover 显村名 + 一句话（`summary`）；点图进详情。
- 视频暂无数据，先纯图；后续有视频再加 tab。

### 5.5 分类浏览 `/villages/tags`（新页）
- 按 `tags` 六大类（文化/自然/产业/荣誉/人文/新业态）聚类逛村。
- 六大类做分区，每类下列带该标签的村庄卡；与详情页标签模块互跳。

## 六、数据与代码影响

- **保留并复用**：`VillageListView.vue`、`VillageDetailView.vue`、`encyclopedia-villages.json`、`lib/encyclopedia.js`。
- **改造/移除**：`modules/people`、`modules/media`、`modules/ranking` 三个模块目录——移除其 View，routes 改为重定向。
- **详情页**：重排模块 + 新增人物占位模块。
- **新增**：荣誉榜单、影像长廊、分类浏览三个 View（建议放在 `modules/villages/` 下，与名录/详情同域），及各自路由。
- **数据文件**：`villages.json` 淘汰（确认无其他引用后删除）。

## 七、非目标（本次不做）

- 不做人物数据的实际录入/采集（仅留占位入口）。
- 不做视频影像（无数据）。
- 不做 Canvas 海报/导览图生成（工具条按钮维持"敬请期待"占位）。
- 不改动首页、其他一级栏目。

## 八、完成标准

- 全站村庄数据统一到 12 全国名村，人物/影像/榜单子页不再引用 6 村老数据。
- 导航下拉三项（荣誉榜单/影像长廊/分类浏览）path 不与 `/villages` 撞车，跳转准确。
- 详情页按新顺序渲染，含人物占位模块，无重复按钮。
- 三个横向页均用真实字段渲染，无"待采集"空壳（人物占位除外，且它在详情页内）。
- 旧路由 `/people`、`/media`、`/ranking` 重定向不 404。
- `npm test` 全绿、`npm run build` 成功。
