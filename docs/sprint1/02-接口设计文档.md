## 1. 通用约定

**Base 路径**：`/api/practice/media`（在 `app.js` 挂载 `makeMediaRouter`）。

**鉴权**：本组全部接口需登录。请求头带 `Authorization: Bearer <JWT>`；中间件 `makeAuth` 解出 `req.user = { id, username }`，失败返回 401。

**归属校验**：凡带 `dossierId` 的接口，都先 `assertDossierMember`——查档案的 `team_id`，再校验当前用户是该队成员，否则：
- 档案不存在 → `404`
- 非本队成员 → `403`

**请求体格式**：
- 文件类接口用 `multipart/form-data`（字段名固定 `file`，附带 `dossierId`）。
- 纯数据接口用 `application/json`（app 层 `express.json({ limit: '1mb' })`）。

**错误响应**：统一经 `errorHandler` 出口，格式 `{ "error": "<提示信息>" }`，HTTP 状态码见各接口。

**恒 200 的接口**：`/extract`、`/summarize`、`/describe-image` 在 AI 不可用时**不报错**，而是返回带 `source`/`available` 标记的空/降级结果（见下）。这是刻意的健壮性设计——AI 失败不该阻断队员采集。

## 3. 接口详情

### 3.1 POST `/media` —— 上传材料

存储任意类型材料并归类。图片/音视频只存不解析。

**请求**（multipart）：`dossierId` + `file`

**响应 `201`**：
```json
{
  "media": {
    "url": "/uploads/practice/<dossierId>/<随机名>.jpg",
    "name": "村口合影.jpg",
    "size": 1048576,
    "ext": "jpg",
    "kind": "image"
  }
}
```
`kind`：`image` / `av` / `doc` / `table` / `other`。

**错误**：`400` 缺 dossierId / 缺文件 · `413` 超该档大小上限 · `404/403` 归属校验失败。

---

### 3.2 POST `/media/extract-text` —— 上传并抽纯文本（不存盘）

只把文档/表格抽成纯文本返回，**不落盘**。用于「只想让 AI 读，不想存档」。

**请求**（multipart）：`dossierId` + `file`

**响应 `200`**：
```json
{ "text": "抽取出的纯文本……", "truncated": false }
```
`truncated`：原文超 2 万字被截断时为 `true`。

**错误**：`422` 文件类型不可解析（非 doc/table）· 其余同上。

---

### 3.3 POST `/media/extract-and-store` —— 存盘 + 抽文本（一趟往返）

文本档既进材料清单（有可查看 url），又同时返回文本供后续 AI 提取。前端 `UploadPanel` 选「文档/表格」行走这条。

**请求**（multipart）：`dossierId` + `file`

**响应 `201`**：
```json
{
  "media": { "url": "...", "name": "调研报告.docx", "kind": "doc", "ext": "docx", "size": 20480 },
  "text": "抽取的纯文本……",
  "truncated": false
}
```

**错误**：`422` 不可解析类型（**不存盘**，避免脏文件）· `413` 超限 · `400/404/403`。

---

### 3.4 POST `/media/extract` —— AI 结构化提取 ⭐

从纯文本里抽人物 / 指标 / 材料要点。**本模块最核心的 AI 接口。**

**请求**（json）：
```json
{ "text": "李大爷是返乡创业带头人，他说……去年销量5000元，今年12000元……" }
```

**响应 `200`**：
```json
{
  "people": [
    { "name": "李大爷", "role": "返乡创业带头人", "quote": "……",
      "story": "2-3句人物故事", "highlight": "返乡创业带头人",
      "confidence": 0.9, "source": "auto" }
  ],
  "metrics": [
    { "name": "季度销量", "value": "12000", "unit": "元", "insight": "较去年增长明显",
      "isHighlight": true, "confidence": 0.85, "source": "auto" }
  ],
  "materialHints": [
    { "name": "访谈录音", "note": "", "summary": "一句摘要", "theme": "产业帮扶",
      "confidence": 0.7, "source": "auto" }
  ],
  "source": "ai"
}
```

**`source` 字段（顶层）标明结果来源**：
- `ai` —— AI 成功提取
- `empty` —— 输入文本为空
- `error` —— AI 调用失败（无 key / 超时 / 网络断），此时三个数组均为 `[]`

**恒 200**：AI 失败也返回 200 + 空结果，前端据 `source` 提示降级，不崩溃。

**错误**（仅参数问题才抛）：`400` 缺文本 · `413` 文本超 2 万字上限。

**字段语义**（见 [extractPrompt.js](../../server/services/prompts/extractPrompt.js)）：
- 事实字段（`name/role/quote/value/unit/note`）忠于原文，无法确定给空串；
- 加工字段（`story/highlight/insight/summary/theme`）基于原文归纳，不得引入原文没有的数字；
- `confidence` 0-1，表示该条抽取把握度；`source:'auto'` 标记为 AI 自动产出（供审校区区分人工/AI）。

---

### 3.5 POST `/media/summarize` —— 成果综述生成

汇总已采集的人物/指标/材料，生成一段成果综述 + 若干亮点。

**请求**（json）：
```json
{
  "people": [...], "metricValues": [...], "materials": [...],
  "topic": "电商帮扶", "village": "小朱湾村"
}
```

**响应 `200`**：
```json
{ "summary": "本次实践……", "highlights": ["亮点1", "亮点2"], "source": "ai" }
```
无数据 / 无 key / 失败时 `source` 非 `ai` 且内容为空。恒 200。

---

### 3.6 POST `/media/describe-image` —— 图片轻量描述

给一张图返回一句图注（多模态）。

**请求**（multipart）：`dossierId` + `file`（仅图片）

**响应 `200`**：
```json
{ "available": true, "description": "村口广场上村民正在……" }
```
模型不支持视觉 / 无 key / 失败：
```json
{ "available": false, "reason": "当前模型不支持图片输入" }
```

> 注：DeepSeek `deepseek-chat` 暂不支持图片输入，接口已就位，换支持视觉的 model/endpoint 即可启用。

**错误**：`422` 非图片文件 · `400/404/403`。

---

### 3.7 POST `/media/import-zip` —— ZIP 整包导入 ⭐

解压 zip，逐文件过「存盘 + 归类 + 文本档 AI 抽取」管线，一次导入一批材料。

**请求**（multipart）：`dossierId` + `file`（.zip，≤100MB）

**响应 `201`**：
```json
{
  "materials": [ { "type": "文档", "name": "报告.docx", "url": "...", "kind": "doc", "ext": "docx", "text": "..." } ],
  "drafts": {
    "people": [ { "name": "...", "sourceFile": "报告.docx", "source": "auto", ... } ],
    "metrics": [ ... ],
    "materialHints": [ ... ]
  },
  "skipped": [ { "name": "损坏.docx", "reason": "无法读取" } ],
  "imported": 8,
  "total": 10
}
```
- `materials`：成功入库的材料；`drafts`：文本档抽取出的待审校草稿（每条带 `sourceFile` 标来源文件）；
- `skipped`：跳过的文件及原因；`imported`/`total`：成功数 / 总条目数。

**安全限额**：解压总量 ≤300MB、有效文件 ≤100 个；自动跳过目录 / `__MACOSX` / 隐藏文件。单个文件存储或解析失败 **不阻断整体**，记入 `skipped` 继续。

**错误**：`400` 非法 zip / 空包 · `413` 文件过多 / 解压总量超限 · `400/404/403`。

## 4. 前端调用封装

`mediaApi.js` 统一封装 multipart 请求（`api.js` 的 `request` 只发 JSON，故独立一套）：

| 前端函数 | 对应接口 |
|----------|---------|
| `uploadMedia(dossierId, file)` | POST `/media` |
| `extractUploadedDoc(dossierId, file)` | POST `/media/extract-text` |
| `extractAndStoreDoc(dossierId, file)` | POST `/media/extract-and-store` |
| `describeImage(dossierId, file)` | POST `/media/describe-image` |
| `importZip(dossierId, file)` | POST `/media/import-zip` |

`postForm` 自动带 `Authorization`，统一把非 2xx 转成带 `status` 的 Error（网络不通 `status:0`），前端据 `413/422` 给不同提示。

## 5. 乡村百科接口（补充）

Base 路径 `/api/villages`。列表/详情/标签**无需登录**（公开数据）；写操作待加管理员鉴权。

| 方法 | 路径 | 用途 | 说明 |
|------|------|------|------|
| GET | `/api/villages` | 分页列表 + 搜索筛选 | 查询参数 `?page&pageSize&q&province&tag` |
| GET | `/api/villages/meta` | 省份列表 + 热门标签 | 供筛选栏；标签按词频 Top 30 |
| GET | `/api/villages/:id` | 单村详情 | 不存在返回 404 |
| POST | `/api/villages` | 创建村庄 | 待加 admin 鉴权 |
| POST | `/api/villages/batch` | 批量导入 | `INSERT OR IGNORE` 幂等，返回 `{inserted,skipped,total}` |
| PUT | `/api/villages/:id` | 更新 | — |
| DELETE | `/api/villages/:id` | 删除 | — |
| POST | `/api/villages/:id/enrich` | AI 内容增强 | 生成 intro/facts/tags；已有人工编辑自动跳过 |
| GET | `/api/villages/:id/enrich/status` | 查询 AI 可用性 | 返回各字段是否存在、来源、`aiAvailable` |

**GET `/api/villages` 响应 `200`**：
```json
{
  "villages": [ { "id": "chenjiapu", "name": "陈家铺村", "province": "浙江省",
    "coord": [119.5, 28.3], "tags": {"文化类":["古村落"]}, "honors": ["中国传统村落"], ... } ],
  "total": 12, "page": 1, "pageSize": 20, "totalPages": 1
}
```
`villages[]` 的 JSON 列（`tags/honors/gallery/facts/...`）在 service 层 `rowToVillage` 解析后返回，前端拿到即用。

**POST `/api/villages/:id/enrich` 响应 `200`**：
```json
{ "villageId": "chenjiapu", "generated": ["intro","facts","tags"],
  "skipped": ["intro"], "results": { "intro": {...}, "facts": {...}, "tags": {...} } }
```
`skipped` 列出因已有人工编辑而跳过的字段——**AI 不覆盖人写的内容**。

## 6. 设计取舍问答（验收备答）

**Q：为什么 `/extract` 等 AI 接口失败也返回 200？**
A：AI 是辅助工具而非主流程。队员在现场采集不能因为 DeepSeek 超时就卡住。用 `source`/`available` 字段告知前端降级，比抛 500 更符合真实使用场景。

**Q：为什么上传要区分 `extract-text`（不存）和 `extract-and-store`（存）？**
A：两种诉求——"只让 AI 读一遍不留档" vs "既留档又抽取"。分开两个端点语义清晰，前端按场景选。

**Q：ZIP 里某个文件坏了会怎样？**
A：单文件失败记入 `skipped` 并 `continue`，不影响其余文件导入。整体只在 zip 本身非法或超限时才整体失败。

**Q：多队成员并发上传会不会打爆 AI？**
A：全局令牌桶单例限流（5次/秒），ZIP 导入对每个文本档触发抽取时也走同一限流器。

**Q：百科列表接口为什么不鉴权？**
A：村庄是公开展示数据，任何访客都可浏览检索；只有写操作（创建/更新/删除/enrich）后续加管理员鉴权。

**Q：AI enrich 会不会把人工写好的介绍覆盖掉？**
A：不会。`enrich` 会先检查 `intro` 是否已有人工编辑（>100 字视为已编辑），是则跳过，仅对空缺字段生成，结果在 `skipped` 里返回。
