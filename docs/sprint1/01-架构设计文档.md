## 1. 模块定位

「我的实践」是一个三阶段工作台，按 `实践前(plan) → 实践中(track) → 实践后(result)` 推进，每阶段一个 Vue 组件。

**实践中**要解决的核心问题：队员在乡村实践现场会产生大量原始材料（访谈录音转写、调研文档、表格数据、照片、音视频），但这些材料是**非结构化**的，无法直接变成成果展示。

本模块把「实践中」从一组静态手工录入表单，升级为 **AI 采集工作台**：

```
队员上传材料  →  文档/表格自动解析出纯文本  →  AI 从文本里结构化提取
（人物 / 指标 / 材料要点）  →  队员审校采纳  →  回填档案 payload
                                            ↓
                          四维进度看板对照方案实时督进
```

一句话：**把「一堆文件」变成「档案里可用的结构化词条」，并在过程中盯住进度。**

## 2. 技术选型与理由

| 层 | 选型 | 为什么这样选 |
|----|------|-------------|
| 前端框架 | Vue 3 + Vite + Vue Router | 组合式 API（`<script setup>`）便于把工作台拆成职责单一的小组件；Vite 冷启动快，开发体验好 |
| 后端框架 | Express 5 | 轻量、路由中间件模型直观；`makeXxxRouter(deps)` 工厂函数便于依赖注入与测试 |
| 数据库 | better-sqlite3（SQLite） | 单文件、零部署，实训环境开箱即用；**同步 API** 写起来无 async 心智负担，事务简单。团队评估后否决了 Postgres（部署成本不匹配实训场景） |
| 文件上传 | multer（内存存储） | 拿到 buffer 后交给 service 层按类型分档校验，multer 层只做最大档硬顶 |
| 文本解析 | mammoth(docx) / pdf-parse(pdf) / xlsx / papaparse(csv) | 按扩展名分派，覆盖实践最常见的文档与表格格式 |
| ZIP 解压 | jszip | 队员常把一次实践材料打包成 zip，支持整包导入省去逐个上传 |
| AI 能力 | DeepSeek Chat（OpenAI 兼容接口） | 强制 `response_format: json_object`，结构化输出稳定；密钥只在服务端读，不暴露给前端 |
| 鉴权 | JWT（jsonwebtoken）+ bcryptjs | 无状态 token，中间件解出 `req.user` 即可；密码只存 bcrypt 哈希 |
| 测试 | Vitest + supertest + @vue/test-utils | 前后端同一测试栈；supertest 直接打 Express app（不监听端口）；解析库/fetch/fs 全部依赖注入便于 mock |

## 3. 分层架构

后端严格分四层，**依赖方向自上而下，AI 只在 service 层**：

```
路由层 routes/media.js
  ├─ 鉴权中间件 makeAuth  → req.user
  ├─ 归属校验 assertDossierMember  → 只有本队成员能给档案传材料
  └─ 参数校验 → 调 service
        │
服务层 services/
  ├─ mediaService.js          纯存储（分档校验 + 分目录写盘）
  ├─ practiceExtractService.js AI 结构化提取（调 DeepSeek + 三级兜底）
  ├─ practiceSummaryService.js 成果综述生成
  ├─ imageDescribeService.js   图片轻量描述
  └─ zipImportService.js       解压 + 逐文件过上面的管线
        │
工具层 lib/
  ├─ fileText.js     按扩展名抽文本（解析库依赖注入）
  ├─ deepseek.js     DeepSeek 薄封装（超时/重试/错误分类）
  ├─ rateLimiter.js  令牌桶限流（全局单例，5次/秒）
  └─ validate.js     httpError 工具
        │
数据层 db/
  └─ better-sqlite3 连接 + schema.sql（dossiers.payload 整存 JSON）
```

**设计原则：**

1. **路由薄、服务厚**：路由只做鉴权 / 校验 / 组装响应，业务逻辑全在 service。
2. **AI 调用集中在 service 层**，工具层 `deepseek.js` 只负责「怎么调通一次请求」，不含业务语义。
3. **依赖注入到处可测**：`extractText(buffer, name, { parsers })`、`storeFile({ ..., fsImpl })`、`chatJSON({ ..., fetch })` 都能注入 mock，单测不真正加载重库 / 写盘 / 联网。

## 4. 一次「上传文档 → AI 提取」的完整数据流

以队员上传一份访谈记录 docx 为例（前端 `UploadPanel.vue` 选「文档」行）：

```
① 前端 UploadPanel.onPick
     └─ mediaApi.extractAndStoreDoc(dossierId, file)   [multipart]
        │
② POST /api/practice/media/extract-and-store
     ├─ makeAuth      解 JWT → req.user
     ├─ fixName       修 multer 的中文名 latin1 乱码
     ├─ assertDossierMember  查 dossiers.team_id + assertMember
     ├─ isParsable    仅 doc/table 放行，否则 422
     ├─ storeFile     分档校验大小 → 按 dossierId 分目录写盘 → 返回 { url,name,kind,ext }
     └─ extractText   docx → mammoth 抽纯文本（超 2万字截断）
        └─ 返回 { media, text, truncated }
        │
③ 前端拿到 media 入材料清单；再把 text 交 AI 提取
     └─ POST /api/practice/media/extract  { text }
        ├─ 令牌桶 acquire（限流）
        ├─ chatJSON → DeepSeek（强制 JSON，20s 超时，失败重试一次）
        └─ 规范化 people/metrics/materialHints，每条标 source:'auto' + confidence
        │
④ 前端并入「待审校区」draft，右栏 Tab 展示；队员逐条采纳/丢弃
     └─ 采纳 → 写入 state（档案编辑态）→ save → PATCH 档案 payload 落库
```

**关键健壮性设计：**

- **上传即解析、一趟往返**：`extract-and-store` 让文件既进材料清单（有可查看 url），又同时返回文本，省一次上传。
- **文件名乱码修复**：multer 按 latin1 解码 `originalname`，中文名会乱码；`fixName` 检测「latin1 化的 utf8」并解回。
- **不可解析类型不存脏文件**：非 doc/table 走 `extract-and-store` 直接 422，不落盘。

## 5. AI 三级兜底（本模块最关键的健壮性设计）

AI 是外部依赖，会无 key、会超时、会返回残缺 JSON。抽取接口 `POST /extract` 的设计原则是 **恒返回 200，绝不因 AI 失败而阻断队员工作**：

| 场景 | source 标记 | 返回 |
|------|-----------|------|
| 空输入文本 | `empty` | 空结果 `{people:[],metrics:[],materialHints:[]}` |
| LLM 调用失败（无 key / 超时 / 网络断） | `error` | 空结果（`console.warn` 记录，不抛） |
| 成功但返回非对象 | `error` | 空结果 |
| 成功 | `ai` | 规范化后的结构化结果 |

前端根据 `source` 提示用户（如"AI 暂不可用，可手动登记"），**降级但不崩溃**。这也是可作为 AI 禁飞区手写讲解的核心逻辑：`practiceExtractService.js` 的 try/catch + `emptyResult(source)` 分支。

`deepseek.js` 再往下兜一层：区分 `NoKeyError`（不重试）与 `DeepseekError`（网络抖动重试一次），20s `AbortController` 超时。

## 6. 限流：令牌桶单例

所有 AI 调用共享一个全局 `TokenBucket`（5 次/秒），`lib/rateLimiter.js`：

- 令牌恒定速率补充，`acquire()` 无令牌时进等待队列；
- 200ms 定时器 `_refill + _drain` 兜底排空队列（`unref` 不阻塞进程退出）；
- ZIP 整包导入会对每个文本档触发一次 AI 抽取，限流器保证不瞬间打爆 DeepSeek 配额。

## 7. 前端组件拆分

`StageTrack.vue` 是实践中工作台的容器，按职责拆成：

```
StageTrack.vue（容器：状态 state + 待审校 draft + 采纳/保存逻辑）
├─ TrackProgress.vue   四维进度看板 + 动态督进（通栏）
├─ 左栏
│   ├─ UploadPanel.vue    按类型分行上传（zip/文档/表格/图片/音视频）
│   ├─ TrackExtract.vue   自然语言输入 + 成果综述
│   └─ TrackMedia.vue     手动登记材料（补料）
└─ 右栏（Tab 切换）
    ├─ 指标 Tab   + DraftReview（待审校）
    ├─ 材料 Tab   MaterialGroups 按类型分组 + DraftReview
    └─ 人物 Tab   + DraftReview
```

**待审校区（draft）提升到容器**：AI 抽取的结果不直接写档案，而是先进 `draft`，队员在右栏各 Tab 逐条「采纳/丢弃」，采纳才写入 `state` 并落库。这保证 **AI 是辅助、人是决策者**——符合实训的 AI 合规要求（AI 产出必须经人工审校）。

`mergeDraft` 做去重（按 name+quote / name+value / name 作 key），避免重复上传同一文档时草稿堆叠。

## 8. 进度看板：四维动态督进

`gapAnalysis.js`（纯函数）对照方案分析缺口，是"督进度"的算法核心：

- **缺口清单**：方案未生成、计划指标未登记 / 只填一半、材料偏少、人物偏少。
- **动态督进 `pace-slow`**：对照档案 `startDate/endDate` 算时段进度 `elapsedRatio`，对照四维（任务勾选率 / 指标填齐率 / 材料数 / 人物数）均值算整体完成度 `progress`。当 **时段过半（>50%）但完成度不足（<50%）** 时，给出 warn 级督进提醒。

纯函数、无副作用，便于单测覆盖各种缺口组合。

## 9. 安全设计

| 关注点 | 措施 |
|--------|------|
| 越权传材料 | `assertDossierMember`：查档案 `team_id`，再 `assertMember` 校验用户是本队成员，否则 404/403 |
| 路径穿越 | `storeFile` 重铸存储名（`genId()` + 原扩展名），不用原始文件名落盘 |
| 静态资源暴露面 | `express.static` 只挂 `uploads/practice`，不暴露其他目录 |
| 上传体积 | multer 硬顶（音视频 200MB / zip 100MB）+ service 层分档细校验 + ZIP 解压总量 300MB / 100 文件上限（防 zip 炸弹） |
| 密钥泄露 | DeepSeek key 只在服务端 `process.env` 读，前端永不接触 |
| 密码 | bcrypt 哈希存储，绝不存明文 |

## 10. 补充板块一：首页 2D 下钻地图

首页（[HomeView.vue](../../src/views/HomeView.vue)）是平台入口，结构：英雄区（轮播背景 + slogan + 搜索框 + 热门词 + 快捷入口）+ 乡村头条跑马灯 + **全国 2D 下钻地图**（[ChinaMap2D.vue](../../src/components/ChinaMap2D.vue)）+ 数据看板 + 模块导航卡。

> 说明：地图早期做过 3D 版（`ChinaMap3D.vue`，echarts-gl），后改为 **2D echarts geo 地图**（更稳、加载快、移动端友好），首页现引用的是 `ChinaMap2D.vue`。`ChinaMap3D.vue` 为遗留组件，首页不再使用。

**下钻逻辑与地图渲染解耦，分三块：**

1. **状态机 `mapDrill.js`（纯函数、不可变）** —— 只管"当前在哪一级"，不碰 DOM：
   ```
   LEVELS = ['country', 'province', 'city', 'district']   // 全国→省→市→区县
   createDrill()          初始栈 [{全国}]
   drillDown(state, tgt)  入栈下一级（未到末级）
   drillUp(state)         出栈上一级
   goToDepth(state, n)    截栈到第 n 层（面包屑点击）
   breadcrumb(state)      各级名字 → 面包屑
   ```
   每次操作返回新 state（`[...stack]` / `slice`），不改原对象，便于单测、便于 Vue 追踪。

2. **geoJSON 加载 `geoLoader.js`** —— 按 adcode 加载对应层级地图数据并缓存；加载失败回调 `onError`，组件置 `loadError` 并**停留在当前层级**（降级不崩）。

3. **渲染组件 `ChinaMap2D.vue`（echarts geo）**：
   - 暖绿纸感配色（对齐《设计风格与配色》），村庄散点用珊瑚橘，选中放大 + 显示标签；
   - 点击区域 → `enterRegion` → `drillDown` 下钻；点击散点 → emit `select-village`；点空白 → 取消选中；
   - 右上角**收纳式搜索面板**：输入关键词搜「当前层级内的村庄 + 可下钻地区」，命中村庄则 `zoom` 放大定位；无关键词时列出当前层级可下钻的下级地区快捷按钮；
   - `ResizeObserver` 自适应尺寸，`onBeforeUnmount` 时 `dispose` 释放。

村庄散点 / 区域过滤由 `lib/villages.js`（`toScatterPoints` / `filterByRegion`）提供，village 数据来自后端 `fetchAllVillages`。

## 11. 补充板块二：乡村百科

乡村百科是村庄数据库的展示端：列表检索页 + 详情页。后端 `villages.js` 路由 + `villageService.js`。

**分层同实践中一致（路由薄、服务厚）**：

```
routes/villages.js   列表/搜索/详情/标签聚合/CRUD/AI增强
services/villageService.js   SQL 查询 + rowToVillage（标量列 + JSON 列解析）
services/aiContentService.js AI 内容增强（生成 intro/facts/tags）
```

**要点：**
- **列表检索**：`list()` 支持分页 + 关键词（name/full_name/summary 模糊）+ 省份 + 标签筛选，命名参数拼 WHERE，防注入。
- **标签聚合 `getMeta()`**：扫所有村的 `tags` JSON，统计词频取 Top 30 热门标签 + 省份 distinct 列表，供筛选栏。
- **AI 内容增强 `/enrich`**：对信息稀疏的村，AI 生成介绍/名片事实/标签。**人工优先**：`intro` 已有人工编辑（>100 字）则跳过，不覆盖人写的内容——与实践中"AI 辅助、人决策"一脉相承。
- **批量导入 `/batch`**：事务内 `INSERT OR IGNORE`，幂等，重复 id 自动跳过，返回 `{inserted, skipped, total}`。

> 百科的村庄数据模型（villages 表标量列 + JSON 列）详见数据模型文档第 7 节。
