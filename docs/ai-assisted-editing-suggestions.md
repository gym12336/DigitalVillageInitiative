# AI 辅助成果编辑 — 功能建议

## 背景

成果搭建台（builder 模块）目前有 4 种基础组件类型（`text`、`image`、`chart`、`agri-sensor`）和 `big-component` 复合组件类型，在组件库中以 17 个语义条目展示，按 14 个叙事场景分类。用户通过拖拽在画布上搭建成果展示页面。

---

## 1. 自然语言生成组件（最高 ROI）

用户说一句话，AI 直接在画布上创建组件。

**示例：**
> "帮我展示李家村 2024 年茶叶产量从 120 吨增长到 210 吨的变化"

→ AI 自动创建一个柱状图组件，填好 `csvText` 和 `title`。

**实现要点：**
- 新增一个 AI 面板（类似 practice 模块已有的 `WebSearchModal`）
- 接收自然语言输入，调用 LLM，返回结构化 JSON：`{ type, props }`
- 走 `stageEditor.js` 的 `addComponent` 流程，与手动拖拽走同一条路径
- 不需要改变任何现有组件模型

**覆盖面：** 文本 + 图表覆盖 17 个组件条目中的 14 个。

---

## 2. 智能文案润色（最容易实现）

text 组件选中时，提供"AI 润色"按钮。

**功能：**
- 扩写 / 缩写 / 改风格（正式 / 亲切 / 数据驱动）
- 保持 `fontSize`、`color` 等样式属性不变，只改 `text` 字段

**实现要点：**
- 在 `PropertyPanel.vue` 中加一个按钮 + 一个 API 调用
- 改动面极小，立竿见影

---

## 3. 数据驱动的图表推荐

用户粘贴一段原始数据（或上传 CSV），AI 分析数据特征后推荐最合适的图表类型。

**推荐逻辑：**
| 数据特征 | 推荐图表 |
|----------|----------|
| 对比型数据 | 柱状图 / 哑铃图 |
| 占比型数据 | 饼图 / 堆叠柱 |
| 时序数据 | 折线图 / 时间轴 |
| 地理数据 | 地图散点 |
| 流向关系 | 桑基图 |

**前置依赖：** 需要先为雷达图、桑基图、日历热力等目前只渲染成柱状图的占位图表实现专属渲染逻辑。

---

## 4. 整页模板生成（远期目标）

> "帮我生成一个李家村的乡村振兴成果汇报页"

AI 根据村庄数据（可从 villages 模块 API 获取）一次性生成完整页面结构。

**生成内容：**
- 封面大图 + KPI 卡组 + 关键变化图表 + 人物故事文本

**依赖：**
- 模板引擎
- 多轮 LLM 调用
- 与 villages 模块的数据对接

---

## 5. AI 排版美化（"一键美化"）

用户摆好组件后，AI 自动优化画布布局。

**功能：**
- **自动对齐：** 检测画布上各组件的 x/y 坐标，按网格或视觉分组自动规整
- **智能间距：** 统一组件间的水平/垂直间距，保证呼吸感
- **配色建议：** 分析现有组件的色板，推荐互补色或统一主题色，一键应用

**实现要点：**
- 输入：画布上所有组件的 `{ id, x, y, width, height, props }`
- 输出：调整后的坐标和样式属性
- 批量调用 `updateComponent`
- 纯计算型任务，不涉及新组件类型

---

## 6. AI 配图助手

解决 image 组件手动填 URL 的痛点。

**功能：**
- **语义搜图：** 用户输入"李家村茶园"，AI 从图库 / Unsplash / 本地素材库匹配
- **图表自动 alt 文本：** chart 组件渲染后，AI 生成描述性 alt 文本（无障碍 + SEO）
- **智能裁剪建议：** 上传大图后，AI 建议合适的 `objectFit` 和裁剪区域

---

## 7. 叙事线索编排（Storyline Builder）

从"搭积木"升级到"讲故事"。

**示例：**
> 输入：李家村、茶叶产业、脱贫成果
>
> AI 输出叙事结构：
> 1. 封面大图（村庄全景）
> 2. KPI 卡组（关键数字：收入、产量、户数）
> 3. 前后对比图（2023 vs 2025）
> 4. 人物卡（茶农老张的故事）
> 5. 金句块（"一片叶子富了一方百姓"）

**特点：**
- AI 只规划结构不填具体内容，用户保留掌控感
- 一键生成骨架（占位内容），用户逐项填入真实数据
- 比「整页模板生成」更轻量

---

## 8. 数据洞察自动标注

chart 组件目前只渲染图形，没有洞察文字。AI 读取 `csvText` 数据后自动生成标注。

**标注类型：**
- **异常标注：** "2024 年 Q3 产量骤降 40%，与当年旱灾吻合"
- **趋势总结：** "三年复合增长率 18.7%，高于全县平均水平"
- **对比结论：** "李家村茶园亩产值已超过邻村 2.3 倍"

**渲染方式：**
- 作为新的 overlays 层渲染在图表上方
- 或单独存为 text 组件、贴在图表旁边

---

## 9. 多模态输入

串联现有的 voice 模块和 practice 模块能力。

**输入方式：**
- **语音建组件：** 对着话筒说"加一个饼图，显示李家村产业占比"，语音转文字后走 NL → 组件 pipeline
- **拍照建组件：** 拍手绘草图或纸质报表，AI 识别图表类型和数据，自动创建 chart 组件
- **URL 导入：** 粘贴新闻报道或政府公报 URL，AI 提取关键数据和引述，生成图表 + 金句块

---

## 10. AI 审校（Pre-publish Review）

成果页发布前的最后一关。

**检查维度：**
- **数据一致性：** 同一数字在不同组件中出现矛盾（KPI 卡说产量 210 吨，柱状图数据写的 180），AI 标出
- **可读性评分：** 文字是否过小、对比度是否足够、信息密度是否过高
- **叙事完整性：** 是否有封面但没结尾、有数据但没解读、有人物但没引述
- **敏感词检查：** 涉及政策表述是否准确

---

## 11. 学习用户偏好

AI 越用越懂你。

**积累维度：**
- **组件使用频率：** 更爱用饼图还是柱状图？偏爱什么字号/配色？
- **页面结构偏好：** 习惯 封面 → KPI → 图表 → 故事 的顺序，还是反过来？
- **dossier 模板复用：** 同一 dossier 下多页成果风格一致，AI 记住第一页的配色和排版，后续页面自动继承

**实现要点：**
- 偏好数据存在 user profile 中（可与 `bigComponentStore` 并置）
- 作为 prompt 的 few-shot context，无需额外训练

---

## 统一架构建议

不管做哪个层次，建议遵循同一模式：

```
用户输入 → AI Panel（Vue 组件）→ POST /api/builder/ai/generate
                                       ↓
                                    LLM（prompt 中注入组件 schema）
                                       ↓
                                   返回 { type, props, layout? }
                                       ↓
                                   stageEditor.addComponent() 渲染到画布
```

**关键设计原则：**
- 让 LLM 输出结构化 JSON，schema 直接用 `componentFactory.js` 里各 `create*Component` 的默认参数结构，天然对齐
- 复用现有的 store 机制：AI 生成的组件和手动拖拽的组件走同一条 add / update / history 路径，撤销重做都免费获得
- 参考 practice 模块已有的 AI 调用模式（`planGen.js`、`retrieval.js` 等）中的 API 交互方式

---

## 优先级建议

| 优先级 | 功能 | 理由 |
|--------|------|------|
| 🔴 P0 | NL → 图表 / 文本组件 | 覆盖 80% 场景，改动最小 |
| 🔴 P0 | 文案润色 | 实现成本极低，立竿见影 |
| 🟡 P1 | 数据洞察标注 | 让图表从"画的"变成"会说话的" |
| 🟡 P1 | AI 排版美化 | 解决用户最痛的"对不齐"问题 |
| 🟢 P2 | 叙事线索编排 | 从搭积木升级为讲故事 |
| 🟢 P2 | AI 审校 | 发布前质量保障 |
| 🔵 P3 | 多模态输入 | 依赖较多，先做语音 |
| 🔵 P3 | 学习偏好 | 需要数据积累，先埋点 |
