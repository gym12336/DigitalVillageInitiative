## 1 · 天地图搜索不符合 v2 契约（提交 `65f5e1b`）

**功能点**：成果搭建台 3D 地图的村庄搜索定位（`tianditu.js`）。

**问了什么**
> "`builder-tianditu.test.js` 有 3 个测试失败：期望 URL 含 `specifyAdminCode:156530000`，实际是 `specify:530000`；还有兼容 status 对象格式、cityKeyword 过滤返回空。搜索为什么返回空、这几个测试到底要什么？"

**AI 给了什么**
- 指出天地图 v2 搜索接口的契约差异：
  1. 行政区限定参数名是 `specifyAdminCode`，值要用 **9 位国标码**（国家码 `156` + 6 位行政区划码），不是旧的 `specify`+6 位。
  2. 成功状态码在 v2 里可能是对象 `status: { infocode: 1000 }`，不只是字符串 `'0'`。
  3. poi 坐标在 `lonlat` 字段（形如 `"99.9,22.5"`），不是分离的 `lon`/`lat`——旧代码按 `poi.lon && poi.lat` 过滤，v2 数据全被滤掉，所以结果为空。

**我改了什么**
- 采纳了契约判断，但**没有照 AI 给的写法直接贴**，而是按项目现有代码风格改：
  - `postStr` 里 `specify: provinceCode` → `specifyAdminCode: '156' + provinceCode`。
  - 成功判定抽成一个 `ok` 布尔：同时接受字符串 `'0'` 和 `Number(status.infocode) === 1000`，再检查 `Array.isArray(data.pois)`。
  - **自己补了一个 `parseLngLat(poi)` 辅助函数**：优先解析 `lonlat`，兼容旧的 `lon`/`lat`，坐标 NaN 的过滤掉。AI 没提兼容旧格式，是我考虑到不破坏其它调用点加的。
- 结果：23 个单测全绿，且没动 `buildMosaicTiles`/`normalizeRegion` 等其它导出。

**反思**：AI 对第三方 API 契约的记忆不一定准，但它能快速指出"v2 和 v1 差在哪"这个方向。我的价值在于——验证它说的对不对（跑测试）、按项目风格落地、补它没想到的向后兼容。

## 2 · StagePlan 测试在 jsdom 下 fetch 报 Invalid URL（提交 `b65a806`）

**功能点**：实践前"联网搜索集成"的组件测试（`mine-stagePlan-web.test.js`）。

**问了什么**
> "这个测试报 `TypeError: Invalid URL: /api/villages?page=1&pageSize=1000`，但测试明明 mock 了 retrieval.js，为什么还会真的发 fetch？"

**AI 给了什么**
- 分析调用链：测试 mock 的是 `retrieval.js`，但 `StagePlan.vue` 的 `onMounted` 还会调 `sources.js` 的 `getRetrievalSources()`，它内部 `fetchAllVillages()` → `fetch('/api/villages?...')`。
- jsdom 测试环境没有页面 origin，无法解析 `/api/villages` 这种相对路径，所以 `new URL()` 抛 Invalid URL。
- 建议：把 `sources.js` 也 mock 掉，返回空数据源。

**我改了什么**
- 采纳"mock sources.js"的方案。**自己确认了 mock 返回的数据形状**——读了 `sources.js` 源码，确保返回 `{ villages, results, demands, guide }` 四个字段，和真实 `getRetrievalSources` 的契约一致，否则 mock 会让别的断言失真。
- 加了注释说明为什么 mock（jsdom 相对路径问题），方便队友以后看懂。
- 结果：8 个用例全绿，无 Invalid URL。

**反思**：这是典型的"mock 不彻底"问题——mock 了直接依赖，漏了间接依赖。AI 帮我快速定位到 `sources.js` 这一环。我坚持了一点：mock 的返回值必须和真实契约对齐，不能为了让测试过就随便返回。

## 6 · AI 结构化提取服务的三级兜底（提交 `8cce1b1`）

**功能点**：把队员粘贴的访谈/调研文字，用 AI 抽成结构化的人物/指标/材料（`server/services/practiceExtractService.js`）。

**问了什么**
> "调 DeepSeek 抽取结构化数据，如果没配 key、超时、或者模型返回的 JSON 是坏的，怎么保证接口不挂、前端还能用？"

**AI 给了什么**
- 建议做多级降级：空输入直接返回空、LLM 调用失败返回空结果、模型返回非对象也当失败处理。
- 建议给结果打上来源标记，让前端知道是 AI 出的还是兜底。

**我改了什么**
- 采纳降级思路，**自己定了三态返回契约**：`source: 'ai'（成功）| 'empty'（空输入）| 'error'（失败兜底）`，前端据此显示不同提示。
- **补了 AI 没提的两点**：① 超长文本先 `slice(0, TEXT_MAX)` 截断再送模型，防止 token 爆掉；② 加了限流桶 `bucket().acquire()`，避免并发把配额打满。
- 每个字段都用 `Array.isArray` 守卫 + `normXxx` 规范化，模型少字段/多字段都不崩。
- 这套逻辑有 `server-practiceExtract.test.js` 用注入的 `chatImpl` 覆盖，不依赖真实网络。

**反思**：AI 给的是"要降级"的方向，但降级要降到什么状态、前端怎么区分、超长和限流这些工程细节是我补的。三态契约让"AI 挂了"变成一个可预期的正常路径，而不是报错。

## 8 · 乡村百科详情页因空对象崩溃（提交 `52c4699`）

**功能点**：乡村百科详情页渲染。

**问了什么**
> "有的村庄详情页一直卡在骨架屏加载不出来，控制台报错，怎么定位？"

**AI 给了什么**
- 分析报错栈，指出是 `manager` 字段为空对象 `{}` 时，模板里访问它的子属性触发渲染异常。

**我改了什么**
- 采纳定位。**自己核对了数据来源**——确认是部分村庄数据的 `manager` 缺字段，不是个例，所以在渲染处加了空值守卫，而不是只改一条数据。

**反思**：AI 读栈定位很快。但"改数据还是改代码"是我的判断——数据层面的缺失会反复出现，守卫加在渲染处才是根治。

## 10 · 首页 3D 下钻地图的状态机（提交 `364af87` / `2067eab`）

**功能点**：首页中国 3D 地图，点击省→市→区县逐级下钻，带面包屑（`src/lib/mapDrill.js`）。

**问了什么**
> "地图要支持全国→省→市→区县四级下钻，还要面包屑回退。这种多级导航的状态怎么管才不乱？"

**AI 给了什么**
- 建议把下钻状态抽成独立的状态机，用一个层级数组表示当前路径，下钻入栈、回退出栈。

**我改了什么**
- 采纳状态机思路，**自己把它做成一组纯函数**：`LEVELS = ['country','province','city','district']`，配 `createDrill / canDrill / drillDown / drillUp / goToDepth / current / breadcrumb`。

- 这样状态逻辑能脱离地图完全单测（`mapDrill.test.js`），也让"下钻"和"渲染"解耦。

**反思**：AI 给的是"用状态机"这个模式，但把它做成无副作用的纯函数、和地图 SDK 彻底分离，是我为了可测性和可维护性做的设计。地图 SDK 很难测，纯函数状态机很好测——这个边界划对了，后面加层级就很稳。

## 13 · 实践进度的动态督进（提交 `164b124`）

**功能点**：实践中工作台，根据时段和完成度动态提醒队员（`src/modules/practice/mine/gapAnalysis.js`）。

**问了什么**
> "想在队员实践进度落后时给个提醒。怎么判断'落后'？"

**AI 给了什么**
- 建议对比已用时间比例和任务完成比例，落后就提示。

**我改了什么**

- 有 `mine-gapAnalysis.test.js` 覆盖。

**反思**：AI 给了"比较两个比例"的方向，但阈值定在哪、什么时候不该催（缺时间基准）、文案给不给具体数字，是我根据"提醒要有用又不烦人"的产品判断定的。

## 总结：我如何使用 AI

1. **定位问题**：AI 擅长快速定位调用链、指出 API 契约差异、排查"功能是不是没了"，省去我逐行翻代码。
2. **我不照抄**：每条建议我都验证（跑测试）、按项目风格落地、补 AI 没想到的（向后兼容、DRY、排序一致性）。
3. **根因归我**：第 2、3 条的 bug 其实是我自己重构留下的疏漏，AI 帮我确认根因，但修法的取舍（抽辅助函数、对齐契约）是我的判断。
4. **验证优先**：每次改完都以"测试全绿"为准，不靠 AI 说"应该没问题"。
